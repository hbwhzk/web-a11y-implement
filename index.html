其它参考文档：

可感知性

--- 信息和用户界面组件必须以可感知的方式呈现给用户

替代文本

为所有非文本内容提供替代文本

  • 非文本内容可通过简短文字说明：
    • 使用aria-label、alt、aria-labelledby为元素添加标签或属性
  • 非文本内容需要较长的文字描述：
    • 使用aria-describedby、longdesc为元素添加较长文字描述
  • 如果非文本内容是一个验证码
    • 提供一段文本替代文字说明图像目的，并且，
    • 确保同一页面中包含其它感官可以接受的验证码形式。
  • 如果非文本内容一个装饰性内容
    • 设置alt为空
    • 在CSS中设置装饰图
  • 其它情况：
    • 如果非文本内容是一个控件，或者接受用户输入，请参考：HTML中的标准控件和链接，了解控件类型及使用方法
    • 如果非文本内容是一个时基媒体，阅读下节内容

案例一：使用alt为img元素添加标签

案例二：使用longdesc给img元素添加大段描述

案例三：使用aria-label为img元素添加标签

案例四：使用aria-labelledby为控件添加标签

				
<!-- alt -->		
<img src="sample/sample_1_1.jpg" alt="天安门">
				
				
				
<!-- longdesc -->
<img src="pic/sample/sample_1_1.jpg" 
alt="" longdesc="example_2.html#example_1">
				
				
				
<!-- aria-label -->
<input type="text" aria-label="搜索" >

时基媒体

为时基媒体提供替代

  • 为预录的纯音频或纯视频提供替代
    • 预录的纯音频：提供内容脚本；
    • 预录的纯视频：提供内容脚本，或者提供音频描述表达内容；
  • 预录的时基媒体提供字幕或者内容脚本
  • 为直播的音频内容提供等效字幕
  • 为预录的纯视频提供音频介绍

案例一：包含有Closed Caption的音视频文件

案例二：包含有情景字幕和交互式字幕的音视频文件

				
<!-- WebVTT格式的字幕 -->
<video poster="mp4/Monty.jpg" controls width="50%">
  <source src="video/Monty.mp4"  type="video/mp4">
  <source src="video/Monty.webm" type="video/webm">
  <track id="track1" label="English" 
  src="tracks/Monty_subs_en.vtt" 
  kind="subtitles" srclang="en" default>
</video>

适应性

创建可用不同方式呈现的内容

  • 信息、结构或关系，可以被辅助技术获取
    • 使用h1 - h6定义标题头
    • 使用ol、ul、dl定义列表或者组织链接
    • 使用caption和summary为数据表格定义标题和给出详细介绍
    • 使用ARIA定义网页区域
    • 使用id、header关联数据表格中的数据单元

注意：如果使用的技术不能被辅助技术获取，应该通过足够的文字信息说明，例如，某段文字通过视觉变化传递信息的不同，应该同时考虑增加文字，用以强调、突出信息的变化。

参考下节内容，了解更容易通过视觉接受信息。

可辨别性

更容易地通过视觉或听觉接受信息

  • 颜色不作为传递重要信息的唯一元素
    • 包含足够的文本线索
    • 配上相应的符号或图案表达含义
  • 避免音频的自动播放
    • 音频播放不超过3秒，或3秒内自动关闭
    • 只在用户激活的情况下播放音频
  • 视觉对比度 4.5：1
    • 提供一个可以切换对比度的控件
  • 无需辅助技术，文本大小可以放大200%
  • 小心使用文本图像

测试工具：Color Contrast Checker

案例：可切换对比度的控件

				
<!-- 足够的文本线索 -->
<label for="lastname" class="required">
姓名 (必填): 
</label>
<input id="lastname" type="text" />
<style type="text/css">
  .required {
	color:red;
  }
</style>

可操作性

--- 用户界面组件和导航必须可操作

键盘可访问

确保所有功能都能通过键盘来操作

  • 内容和功能可被键盘操作
  • 对每次键击没有时限要求
  • 没有键盘陷阱
    • 能够通过键盘的tab键导航整个页面
    • 键盘焦点不会陷入页面中的某个组件或插件

参考：HTML中的标准控件和链接

				
<!-- 键盘可访问 1 -->	
<a onmouseover="swapImage('hours','','hours_on.gif',1)" 
onfocus=swapImage('hours','','hours_on.gif',1)" 
onmouseout="swapImgRestore()" 
onblur="swapImgRestore()" href="#">
<img alt="hours" src="hours_off.gif" border="0" name="hours">
</a>
				
				
				
<!-- 键盘可访问 2 -->					
<img onclick="nextPage();" onkeypress="nextPage();"  src="arrow.gif"  alt="下一页">

充足的时间

为用户提供足够的时间用以阅读和使用内容

  • 如有内容设置有时间限制，确保时间限制是可以关闭、调整、延长
  • 移动、闪动、滚动、自动更新的信息，可能影响用户使用网页内容，这类信息应该可以被暂停、停止、隐藏

癫痫

不要设计会导致癫痫发作的内容

  • 确保没有任何组件会在1秒钟内闪动3次
  • 尽可能保证闪动区域在较小的范围内

测试工具：PEAT

可导航性

提供帮助用户导航、查找内容、并确定其位置的方法

  • 提供机制绕过重复模块
  • 提供清晰的网页标题，即title
  • 能够通过键盘顺序导航
  • 提供多种方式帮助用户查找到某一个页面
    • 网站地图
    • 搜索
    • 目录
  • 标题（H1）和标签（label）具有描述性
  • 键盘焦点可以视觉呈现

案例一：通过tabindex顺序导航

			
<!-- 绕过重复模块 -->
<div id="skiptodir" class="skiptodir">
<a href="#directory">跳至目录</a>
</div>	
			
			
			
<!-- CSS用于键盘焦点的视觉呈现 -->
a:hover,input:hover {
  text-decoration: underline;
}
a:focus,input:focus {
	outline: 2px solid #e3de7b;
}

可理解性

--- 信息和用户界面操作必须是可理解的

可读性

文本内容可读，可理解

  • 使用lang属性
  • 局部语言，可以被用户代理识别

案例：lang和abbr标签

				
<!-- 使用 lang -->			
<html lang="zh-CN">
  <head>
   <meta charset="UTF-8">
   <title>Web无障碍的编码技巧</title>
  </head>

可预测性

让网页以可预见的方式呈现和操作

  • 用户界面组件被聚集时，不会产生内容或上下文的改变
  • 提供一个按钮以执行动作
  • 重复组件保持相同顺序导航机制
  • 一致功能的组件能够被一致性识别

辅助输入

帮助用户避免和纠正错误

  • 定义错误并以文本形式告知
  • 为需要用户输入的内容提供标签和说明
  • 提供建议性的修正文本
  • 尽可能的预防错误
    • 涉及法律、财务：在递交表单前，用户有机会修改或检查错误
    • 信息删除：用户有机会恢复删除的信息，或要求用户再次确认该动作
    • 考试：用户在递交前，有机会修改或检查错误

鲁棒性

--- 内容必须健壮到可信地被种类繁多的用户代理(包括辅助技术)所解析

兼容

--- 最大化兼容当前和未来的用户代理(包括辅助技术)

  • 规范使用HTML
  • 确保页面能够被正确解析
    • 标签都有正确的闭合
    • id是唯一的
    • 确保元素没有冗余的属性
  • 可被辅助技术使用，并被用户代理接受
    • HTML
      • 参考：HTML中的标准控件和链接
      • 使用label元素关联表单控件文本标签
      • 表单控件的不能使用label元素，使用title属性
    • 脚本语言或其它语言在HTML中重构的控件
      • 通过aria-labelledby提供控件名称
    • 控件通过编程语言自绘
      • 通过WAI-ARIA的role让控件可被辅助技术使用
      • 通过aria-labelledby提供控件名称

查验代码可用性：W3C Markup Validation Service

				
<!-- aria-labelledby提供控件名称 -->				
<input name="searchtxt" type="text" aria-labelledby="searchbtn">
<input name="searchbtn" id="searchbtn" type="submit" value="搜索">
